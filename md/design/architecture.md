# Architecture

## Crate layout

| Crate | Role |
|-------|------|
| `sage-ir` | Main crate: CST, symbols, type representation, checking, resolution, memmap |
| `sage-stash` | Arena allocator: `Stash`, `Ptr<T>`, `Slice<T>`, `Stashed<T>`, derive macros |

`sage-ir` defines the salsa database trait (`sage_ir::Db`) and all
tracked structs/functions. `sage-stash` is a pure data-structure crate
with no salsa dependency (salsa integration is behind a feature flag).

## Module map

```
sage-ir/src/
  lib.rs            — Db trait, module declarations
  symbol.rs         — Symbol (wrapper-of-enum), define_kind_symbols! macro
  scope.rs          — ScopeSymbol, LocalCrateSymbol
  ty.rs             — Ty enum, Binder<T>, FnSig, StructSig, StructFields
  ty_fold.rs        — Type substitution/folding
  span.rs           — AbsoluteSpan, RelativeSpan
  name.rs           — Name (salsa interned string)
  generic_param.rs  — GenericParam, AstGenericParam
  ribs.rs           — Ribs, RibEntry (lexical scope stack)
  source.rs         — SourceFile (salsa input)

  cst/              — Concrete Syntax Tree (per-item stash-allocated)
    fns.rs          — FnCstData, ParamCst
    structs.rs      — StructCstData, FieldCst
    ty.rs           — TypeCst, TypeCstKind, TypeCst::check
    paths.rs        — PathCst, Resolution, PathCst::resolve
    generics.rs     — GenericParamCst, CheckGenerics trait
    expr.rs         — ExprCst (body expressions)
    attrs.rs        — AttrCst

  local_syms/       — Per-kind tracked structs with sig/body queries
    fns.rs          — LocalFnSym (sig, body)
    structs.rs      — LocalStructSym (sig, fields)
    enums.rs        — LocalEnumSym
    mods.rs         — LocalModSym
    traits.rs       — LocalTraitSym
    ...

  check/            — Checking infrastructure
    sig.rs          — Check context (signature lowering)
    body.rs         — BodyCheck context (body checking + inference)
    infer/          — Type inference engine
      egraph.rs     — VersionedEGraph (union-find with versioning)
      skeleton.rs   — Skeleton decomposition for structural unification
      version.rs    — Version tree, Universe, VarInfo
      runtime.rs    — Runtime (wake/sleep for deferred constraints)
      bound.rs      — Bound<'db> (None, AtLeast, Exactly)

  resolve/          — Name resolution
    mod.rs          — Resolver, Namespace, SourceRoot, resolve_name

  memmap/           — Module MEM-map (Minimal Expanded Members)
    mod.rs          — expanded_module query, ExpandedModule
    data.rs         — MemmapEntry, MacroUse, Expansion, MacroCallee
    seed.rs         — Seed entries from parsed items
    expand.rs       — Macro expansion fixpoint loop

  tytree/           — Typed tree (output of body checking)
```

## Data flow

```
Source text
  |
  v
parse_source_file (tree-sitter -> Vec<ItemAst>)
  |
  +---> lower_fn / lower_struct / ... -> per-item CST (Stashed<Ptr<FnCstData>>)
  |
  +---> expanded_module (seed + macro expand) -> MEM-map (Stashed<Slice<MemmapEntry>>)
          |
          v
        Resolver::resolve_segments (name lookup across modules)
          |
          v
  LocalFnSym::sig(db)     LocalStructSym::sig(db)
    |                        |
    | Check context          | Check context
    | TypeCst::check         | TypeCst::check
    |                        |
    v                        v
  Stashed<Binder<FnSig>>  Stashed<Binder<StructSig>>
    |                        |
    v                        v
  LocalFnSym::body(db)     LocalStructSym::fields(db)
    |                        |
    | BodyCheck context      | Check context
    | ExprCst::check         | TypeCst::check
    | inference              |
    v                        v
  TyBody                   Stashed<StructFields>
```

## The salsa layer

**Inputs:**
- `SourceFile` — path + content string
- `SourceRoot` — collection of source files for a crate

**Tracked structs (per-item, stable identity):**
- `LocalFnSym`, `LocalStructSym`, `LocalEnumSym`, `LocalModSym`, ...
- `AstGenericParam`
- `MacroInput`

**Tracked functions (memoized queries):**
- `LocalFnSym::sig`, `LocalFnSym::body`
- `LocalStructSym::sig`, `LocalStructSym::fields`
- `expanded_module` (MEM-map computation, with cycle recovery)

**Interned:**
- `Name` — string interning
- `SymExt` — external symbol handle (CrateNum + DefIndex)

## Symbol system

`Symbol<'db>` is a `Copy` wrapper-of-enum generated by the
`define_kind_symbols!` macro. Each variant holds either a local
tracked struct (`LocalFnSym`, `LocalStructSym`, ...) or an external
handle (`SymExt`). The macro also generates per-kind enums
(`FnSymbol`, `StructSymbol`) with `Ast`/`Ext` variants.

```rust
define_kind_symbols! {
    pub struct Symbol<'db> { data: SymbolDataPriv<'db> }
    pub enum SymbolData<'db> { .. }

    pub enum FnSymbol<'db> { Ast(LocalFnSym<'db>), Ext(SymExtKind::Fn) }
    pub enum StructSymbol<'db> { Ast(LocalStructSym<'db>), Ext(SymExtKind::Struct) }
    ...
}
```

## Testing strategy

Tests live in `crates/sage-ir/tests/`. They construct a `salsa::Database`
with a noop `TcxDb` implementation, create `SourceFile` inputs, run
queries, and assert on the output. The noop tcx returns empty results
for all external-crate queries, allowing signature and body tests to
run without real rustc metadata.
